Methodology

This project trains a motor-imagery EEG classifier on the MOABB Cho2017 dataset. The workflow is split into data preparation, spectral inspection, model training, and evaluation.

Main modules:

- moabb_data.py prepares raw, PyGEDAI-cleaned, and downsampled EEG tensors.
- moabb_train.py trains and evaluates the EEG classifier.
- psd.py plots PSDs for raw tensors.
- psd_pygedai.py plots PSDs for PyGEDAI-cleaned tensors.
- downsampled_psd.py plots PSDs for downsampled tensors.

Data Pipeline

1. Dataset Source

The data source is the Cho2017 motor imagery dataset from MOABB.

- Dataset: Cho2017
- Task: motor imagery
- Classes: left_hand and right_hand
- Original sampling frequency assumption: 512 Hz
- EEG epochs are shaped as (epochs, channels, timepoints)

The task targets motor-imagery rhythms, especially mu and beta activity over sensorimotor regions.

2. Raw Cache

moabb_data.py configures MNE/MOABB to use the local mne_data directory, downloads each subject, extracts EEG epochs and labels, and saves one cache file per subject:

raw/subject_<id>.pt

Each file contains:

(X_sub, y_sub)

where X_sub is EEG data and y_sub contains the class labels.

Subjects are capped at 200 epochs by default. If a subject has more than 200 epochs, only the first 200 are kept.

3. PyGEDAI Preprocessing

After raw caching and truncation, moabb_data.py applies PyGEDAI cleaning.

The script builds a reference covariance from Cho2017 channel locations by:

- Loading subject 1 from Cho2017 to obtain EEG channel names.
- Loading MNE's standard_1005 montage.
- Matching Cho2017 channel names to standard_1005 coordinates.
- Passing a channel coordinate DataFrame to pygedai.interpolate_ref_cov.

Each epoch is passed to PyGEDAI with:

- epoch_data shaped as (channels, timepoints)
- sfreq = 512 Hz
- leadfield = the interpolated reference covariance

The cleaned tensor is read from:

out["cleaned"]

Cleaned subject files are saved to:

pygedai_processed/subject_<id>.pt

If pygedai_processed/subject_<id>.pt already exists, that subject is skipped before expensive PyGEDAI processing starts. This allows interrupted preprocessing runs to resume from the next missing subject.

4. Downsampling

PyGEDAI-cleaned tensors are downsampled from pygedai_processed into downsampled using scipy.signal.resample_poly:

resample_poly(X_sub, up=1, down=4, axis=-1)

This is polyphase resampling along the time axis. With the 512 Hz sampling assumption, the effective downsampled sampling rate is:

512 / 4 = 128 Hz

The downsampling function skips tensors with fewer than 400 timepoints to avoid repeatedly shrinking files that were already downsampled.

5. PSD Inspection

PSD helper scripts use scipy.signal.welch and average power spectral density across epochs and channels.

- psd.py reads raw/subject_*.pt and writes raw_psd/subject_<id>_psd.png.
- psd_pygedai.py reads pygedai_processed/subject_*.pt and writes pygedai_processed_psd/subject_<id>_psd.png.
- downsampled_psd.py reads downsampled/subject_*.pt and writes downsampled_psd/subject_<id>_psd.png.

Raw and PyGEDAI PSD plots assume 512 Hz. Downsampled PSD plots assume 128 Hz.

Training Data Loading

moabb_train.py reads from downsampled by default:

downsampled/subject_*.pt

Files are sorted numerically by subject id. The selected subjects are concatenated along the epoch dimension.

Labels are encoded with numpy.unique rather than sklearn. The input tensor is converted to float32 and reshaped to:

(trials, 1, channels, timepoints)

The extra dimension is the convolution input channel dimension.

Normalization

Before training, EEG data is standardized per epoch and per EEG channel:

X = (X - mean_over_time) / std_over_time

This makes amplitude scale more consistent across subjects and channels.

Dataset Split

The dataset uses a stratified trial-level split with a fixed seed:

- 70% training
- 20% validation
- 10% testing

The split preserves class balance across train, validation, and test sets.

Important note: the split is trial-level, not subject-level. Trials from the same subject can appear in different splits. This is useful for debugging and model development, but subject-independent generalization should eventually be measured with a subject-level or leave-one-subject-out split.

Train-Time Augmentation

Only the training subset receives augmentation. Validation and test tensors are not augmented.

Default augmentations:

- Random temporal shift up to 16 samples.
- Additive Gaussian noise with standard deviation 0.03 after normalization.
- Random channel dropout with probability 0.1.

These augmentations are meant to improve robustness to timing jitter, sensor noise, and channel variation.

Model Architecture

The model is EEGTransformer, a CNN plus transformer classifier with EEG-specific inductive bias.

1. Input

The model receives:

(batch, 1, channels, timepoints)

2. Multi-Scale Temporal Front End

The first stage uses parallel temporal convolution branches with kernel sizes:

- 15
- 31
- 63

Each branch learns temporal filters at a different scale. The branch outputs are concatenated along the feature-channel dimension.

3. Depthwise Spatial Filtering

After temporal filtering, a depthwise spatial convolution spans all EEG channels:

kernel_size = (n_channels, 1)
groups = temporal_channels

This is EEGNet-style spatial filtering. It lets each temporal filter learn its own spatial pattern across electrodes.

4. Separable Temporal Block

The model then applies:

- depthwise temporal convolution
- pointwise 1x1 convolution into d_model features
- BatchNorm
- ELU
- squeeze-and-excitation channel attention
- average pooling
- dropout

This block extracts compact temporal features before transformer encoding.

5. Positional Embedding

The CNN output is reshaped into:

(batch, sequence_length, d_model)

A learned positional embedding is added to preserve time-order information.

6. Transformer Encoder

The default transformer settings are:

- d_model = 64
- n_heads = 4
- n_layers = 2
- feedforward dimension = d_model * 4
- activation = GELU
- dropout = 0.4
- batch_first = True

The transformer is intentionally shallow to reduce overfitting on small EEG datasets.

7. Attention Pooling

Instead of using a CLS token, the model learns attention weights over all temporal tokens and computes a weighted average representation.

8. Classification Head

The pooled representation is passed through:

- LayerNorm
- Dropout
- Linear(d_model, n_classes)

For Cho2017, n_classes is 2.

Training Procedure

Training uses:

- Loss: class-weighted soft cross entropy
- Label smoothing: 0.05 by default
- Mixup: alpha = 0.2 by default
- Optimizer: AdamW
- Learning rate: 0.001 by default
- Weight decay: 1e-3 by default
- Scheduler: CosineAnnealingLR
- Gradient clipping: 1.0 by default
- Default epochs: 20 from the CLI

Class weights are computed from the full loaded dataset. Cho2017 subjects are currently balanced in this cache, so the weights are usually 1.0 and 1.0.

During each epoch:

1. The model is set to train mode.
2. Training batches are moved to GPU if available, otherwise CPU.
3. Labels are converted to smoothed one-hot targets.
4. Mixup optionally combines examples and soft labels.
5. The model produces class logits.
6. Soft cross entropy is computed.
7. Gradients are backpropagated.
8. Gradients are clipped.
9. AdamW updates weights.
10. The cosine scheduler steps once per epoch.

Validation

Validation runs every --eval-every epochs. The script tracks:

- training loss
- validation loss
- validation accuracy
- best validation checkpoint

The best checkpoint is saved to:

best_model.pt

or the path passed through --checkpoint-path.

The loss plot is saved to:

loss.png

Final Evaluation

After training, moabb_train.py reloads the best validation checkpoint and evaluates it on the held-out test set. It prints:

Final Test Accuracy

Scaling-Law Experiment

Unless --skip-scaling is passed, the script trains fresh models on increasing fractions of the training set:

- 12.5%
- 25%
- 50%
- 100%

Each model is evaluated on the same validation set, and the plot is saved to:

scaling_law.png

Useful CLI Options

- --data-dir: folder of subject_*.pt files; defaults to downsampled.
- --num-subjects: number of subjects to load.
- --batch-size: training batch size.
- --epochs: base training epoch count.
- --lr: AdamW learning rate.
- --weight-decay: AdamW weight decay.
- --label-smoothing: smoothing used for classification targets.
- --mixup-alpha: mixup beta distribution alpha; set 0 to disable mixup.
- --grad-clip: gradient clipping norm.
- --no-augment: disable train-time augmentation.
- --noise-std: augmentation noise level.
- --time-shift: max augmentation time shift in samples.
- --channel-dropout: augmentation channel dropout probability.
- --continue-until-improves: continue after --epochs until validation accuracy improves, bounded by --max-epochs.
- --patience: early stop after this many epochs without validation improvement.
- --skip-scaling: skip the scaling-law experiment.

Current Caveats

- The split is trial-level, so validation/test performance can be optimistic compared with subject-independent evaluation.
- The pipeline assumes downsampled data was created with ratio 4 from 512 Hz.
- PSD folders and data cache folders are generated artifacts and are ignored by git.
